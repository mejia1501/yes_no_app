# Yes/No App – Tu Asistente de Decisiones Instantáneas

## Introducción

**Yes/No App** es una aplicación interactiva diseñada para ayudarte a tomar decisiones rápidas con un simple botón. Basada en la serie de tutoriales disponibles en [esta playlist de YouTube](https://www.youtube.com/playlist?list=PLCKuOXG0bPi0sIn-nDsi7ma9OV6MEMkxj), la aplicación responde automáticamente a preguntas cerradas (de tipo sí/no) y presenta respuestas animadas y dinámicas para una experiencia de usuario atractiva. Es perfecta para quienes buscan una herramienta ligera y divertida para resolver dudas cotidianas.

## Características Principales

- Generación de respuestas aleatorias tipo Sí/No (y variantes como "quizás", "pregunta después").
- Interfaz simple, intuitiva y minimalista.
- Animaciones visuales para mayor dinamismo y atractivo.
- Integración de respuestas usando APIs externas para mayor variedad.
- Compatible con dispositivos móviles y de escritorio.
- Código base entendible y modular, ideal como ejemplo educativo de aplicaciones modernas.

## Instalación y Ejecución

Sigue estos sencillos pasos para clonar y ejecutar el proyecto localmente:

```bash
# Clona el repositorio
git clone https://github.com/mejia1501/yes_no_app.git
cd yes_no_app

# Instala las dependencias (ajustar a tu gestor de paquetes)
npm install

# Ejecuta la aplicación en modo desarrollo
npm start
```

*Si usas otro gestor de dependencias (como `yarn` o `pnpm`), adapta los comandos según tus necesidades.*

## Uso

1. Abre la aplicación en tu navegador tras ejecutarla.
2. Haz una pregunta cerrada (sí/no) y presiona el botón principal.
3. Observa la animación y recibe la respuesta de forma instantánea.

**Ejemplo rápido:**

- Usuario: "¿Debo salir hoy?"
- Presiona el botón, la app responde y muestra una animación.

## Tecnologías Utilizadas

- **JavaScript / TypeScript** (según el tutorial original)
- **React** (Framework principal para la interfaz de usuario)
- **HTML5 y CSS3** (Diseño responsivo y animaciones)
- **APIs Externas** (para generación de respuestas dinámicas)
- **Herramientas de gestión de paquetes:** npm, yarn o pnpm
